## correctness findings

- `dummyindex/export/graph.py:32-35` ‚Äî **high** ‚Äî On a cache-hit run every edge's `confidence_score` silently collapses to the `1.0` default, so the committed `symbol-graph.json` is NOT byte-identical to a cold (cache-miss) run of the same source. ‚Äî `_CONFIDENCE_SCORE_DEFAULTS` is keyed by `ConfidenceLevel` enum members (`export/common.py`), but a cache round-trip (`cache.py` save‚Üí`json.loads`) turns `confidence` into a plain string `"EXTRACTED"`/`"INFERRED"`; `dict.get("EXTRACTED", 1.0)` misses the enum key and returns `1.0` for every edge regardless of true confidence. Independently confirmed empirically in the repo's own `.context/features/extraction-pipeline/concerns.md`. Triggering input: run ingest once (populates `.context/cache/{hash}.json`), then re-run ‚Äî the second `symbol-graph.json` differs from the first. ‚Äî Coerce `conf` through `ConfidenceLevel(conf)` before the lookup, or re-key `_CONFIDENCE_SCORE_DEFAULTS` by the uppercase string `.value`.

- `dummyindex/analysis/cluster.py:103-104` ‚Äî **high** ‚Äî Community IDs are NOT stable run-to-run despite the docstring's explicit "Community IDs are stable across runs" guarantee, producing spurious diffs in the committed `symbol-graph.json` `community` field. ‚Äî `final_communities.sort(key=len, reverse=True)` is a stable sort keyed only on length; two equal-size communities keep `raw.values()` insertion order, which is partition-dependent. Compounded: the graspologic `leiden(G)` call at `cluster.py:31-41` passes no `random_seed` (the Louvain fallback at `:48` does seed `42`), so on a graspologic-equipped machine partition order ‚Äî and thus IDs ‚Äî varies, and differs from a Louvain-only machine. Documented in `.context/features/extraction-pipeline/concerns.md`. Triggering input: any graph with ‚â•2 equal-size communities, re-clustered. ‚Äî Add a deterministic secondary sort key (e.g. `key=lambda c: (-len(c), sorted(c)[0])`) and pass a fixed `random_seed` to graspologic `leiden`.

- `dummyindex/export/graph.py:37-38` ‚Äî **medium** ‚Äî `to_json` writes `symbol-graph.json` with `indent=2` and no `sort_keys` / no pre-sort, unlike every other artifact writer in the repo (`manifest.py:189`, maps/meta writers all sort), so node/link order rides entirely on NetworkX internal iteration order with no guard. ‚Äî Combined with the unseeded-Leiden + equal-size-tiebreak issue above, any reorder upstream silently churns the committed graph with no test catching it. Confirmed in `concerns.md` ("no sort guarding it"). ‚Äî Sort `data["nodes"]` by `id` and `data["links"]` by `(source, target, relation)` before `json.dump`, and pass `sort_keys=True`.

- `dummyindex/cli/query.py:32-62` ‚Äî **low** ‚Äî A trailing `--top-k` / `--budget` flag with no value is silently swallowed into the query string instead of erroring, returning a wrong-but-exit-code result. ‚Äî The branch `elif a == "--top-k" and i + 1 < len(rest)` requires a following token; when `--top-k` is the last arg the guard fails and the token falls through to `else: leftover.append(a)`, so `query auth --top-k` searches for the literal words `auth --top-k`. Inconsistent with the explicit "must be an integer" error the same handler raises when a value IS present. Triggering input: `dummyindex context query auth --top-k`. ‚Äî Add an `elif a in ("--top-k", "--budget")` catch-all after the value branches that emits `usage_error`/exit 2 for a missing value.

- `dummyindex/context/hooks.py:184` ‚Äî **low** ‚Äî Dead code: `_SELF_GATE_LINE = _SILENT_GATE` is defined but never referenced anywhere in the package. ‚Äî `grep -rn "_SELF_GATE_LINE" dummyindex/` returns only the definition site; the guard-insertion logic uses `_GATE_VARIANTS` (line 185) instead. Not a behavior bug, but a misleading leftover next to live gate constants. ‚Äî Remove the unused binding.

- `dummyindex/context/build/git_delta.py:225-229` ‚Äî **info** ‚Äî `_parse_diff_line` truncates a path containing a literal tab to its last tab-segment (`parts[-1]`), so such a file would be mis-attributed in drift. ‚Äî `line.split("\t")` then `parts[-1]`; a committed file literally named `a\tb.py` yields path `b.py`. Practically near-impossible: git emits `core.quotePath` quoting for control chars and `_changed_via_git` already passes `-c core.quotePath=false` only for `status`, while name-status diff output quotes tabbed names ‚Äî so the real trigger is exotic. Flagging for completeness, not action. ‚Äî If hardening: split on first tab only (`line.split("\t", 1)`) after stripping the status field.

- `dummyindex/context/build/manifest.py:133-134` ‚Äî **info** ‚Äî `read_manifest` uses unguarded `v["sha256"]/v["size"]/v["mtime"]` so a hand-corrupted manifest entry missing a key raises `KeyError`. ‚Äî Direct dict indexing in the comprehension. NOT critical (the agent that surfaced it overstated it): the manifest lives in the gitignored `.context/cache/` and is written only by `write_manifest` in this same module, which always emits all three keys ‚Äî the only path to a missing key is manual corruption, and a `KeyError` there simply aborts a rebuild (no data loss). ‚Äî Optional: `.get(...)` with a treat-as-changed fallback so a corrupt cache degrades to a full rebuild rather than a traceback.
