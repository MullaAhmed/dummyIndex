## architecture findings

### Bug 1 — pre-existing CLAUDE.md / .claude reconciliation seam

- `dummyindex/installer/install.py:271, 236` — **high** — the auto-init path that runs on every fresh `dummyindex install` folds a pre-existing root `CLAUDE.md` nowhere — it only writes `.claude/CLAUDE.md` and never strips/relocates a legacy `<root>/CLAUDE.md`. — evidence: `_auto_init_project` imports and calls `bootstrap_claude_md(project_root / ".claude" / "CLAUDE.md")` (lines 236, 271) but never imports `migrate_claude_md_location`; that migrator (the only code that detects + relocates + strips a root CLAUDE.md) lives in `cli/migrate.py:73` and is reachable solely from `refresh-indexes` (`cli/refresh.py:5,32`), a command a fresh install never invokes. — suggested fix: have `_auto_init_project` (both branches) call `migrate_claude_md_location(project_root)` instead of / before `bootstrap_claude_md`, so the install path owns the "fold the old CLAUDE.md" responsibility.
  — [round1: defend] status=confirmed: same root as C1/A2 — a single fold-seam (the extracted helper of A3) called by both first-build and install closes all three; this is the install-side caller. Confirmed boundary gap, not a pragmatic shortcut.

- `dummyindex/context/build/runner.py:262-263` — **high** — `build_all(bootstrap=True)`, the shared first-build engine behind `init`/`ingest`, has the same gap: it bootstraps `.claude/CLAUDE.md` directly and never reconciles a pre-existing root `CLAUDE.md`. — evidence: line 263 calls `bootstrap_claude_md(out_root / ".claude" / "CLAUDE.md")`; `migrate_claude_md_location` is not imported in this module (only `bootstrap_claude_md` at line 19). A user with a hand-written root `CLAUDE.md` who runs `dummyindex context init` keeps a dangling root file plus a new managed `.claude/CLAUDE.md` — two sources of truth. — suggested fix: route the first-build bootstrap through `migrate_claude_md_location(out_root)` so strip-and-relocate is the single bootstrap entry point, not a refresh-only afterthought.
  — [round1: defend] status=confirmed: this is the canonical caller of the Bug 1 seam — `runner.build_all` is the shared first-build engine, so the extracted helper landed here covers `init`/`ingest`/`install` alike. Confirmed; verdict on the seam below is "one seam, two call-sites".

- `dummyindex/cli/migrate.py:73-138` — **medium** — the "fold a pre-existing CLAUDE.md into the managed setup" responsibility is misplaced: it sits in a `cli/` module named for *legacy layout migration* and is documented as "Called by the ``refresh-indexes`` command" (`migrate.py:3`), so no install/onboard module owns it. — evidence: only callers are `migrate_legacy_layout` (same file, line 70) and `cli/refresh.py:32`; `onboard.py` and `install.py` never touch it. This is wire-shaped reconcile logic stranded in the migration corner. — suggested fix: extract `migrate_claude_md_location` into a `context/output/` (domain) helper that both first-build (`runner.build_all`) and migration (`refresh-indexes`) call, so the seam is owned once and reused, per `folder-organization.md:30-36` (logic belongs under `context/`, `cli/` stays wire-only).
  — [round1: defend] status=confirmed: this is THE seam the panel converges on. C1/A1/A2/C2 are all the SAME bug viewed from four call-sites (install, first-build, migrate-early-return). The fix is one extracted domain helper with a single contract — "given a project root, fold any root CLAUDE.md (managed-block OR not) into `.claude/CLAUDE.md`, strip the root, return a structured result" — called by `runner.build_all` (covers init/ingest/install) and `refresh.py`. C2's early-return bug (no-managed-block file not folded) is a *correctness* defect that the same extraction must fix in the helper body. One seam.

- `dummyindex/cli/migrate.py:53-66,100,116` — **low** — domain-ish reconcile logic emits via `print(... file=sys.stderr)` rather than returning a structured result, so install/onboard can't surface "relocated your CLAUDE.md" through their own reporting channel. — evidence: lines 53-66 / 100 / 116-138 print directly; this conflicts with the wire-only CLI split (`folder-organization.md:30-36`) and forces any new caller to inherit stdout side-effects. — suggested fix: return a small result dataclass from the relocation helper and let the CLI layer print, matching `hooks.HookResult` / `default_plugins` reporting.
  — [round1: defend] status=confirmed: ties directly into the A3 extraction — the helper MUST return a structured result (not print) precisely so the new install/first-build caller can surface "relocated your CLAUDE.md" through its own channel. Without this, A1/A2's new caller inherits stderr side-effects. Real layering violation per `folder-organization.md:30-36`, not a nit.

### Bug 2 — Stop-hook → reconcile gate seam

- `dummyindex/context/reconcile_gate.py:284-295, 229-238` — **high** — on a freshly-installed index the gate is structurally blind: commit-anchored signals require an anchor that only the agent can create, and the mtime fallback is suppressed the moment an anchor exists, leaving a window where real source drift is never gated. — evidence: `_gate_relevant` (284-295) blocks on `unassigned_new_files`/`awaiting_enrichment` OR (mtime `rows` AND `not _has_live_anchor`). A brand-new install has `indexed_commit is None` (only the agent stamps it — `reconcile_gate.py:17-18`), so `compute_reconcile_report` returns early with empty `unassigned`/`drifted` (`build/reconcile.py:128-134`); `_has_live_anchor` is False (235-238) so mtime *could* fire — but a just-built index has fresh mtimes, so nothing is stale yet. The first session that edits source therefore produces drift the gate cannot see until an anchor is stamped, which never happens automatically. — suggested fix: when `indexed_commit is None`, treat the gate's anchor-less branch as authoritative on mtime (already does) AND surface a one-time "stamp your first anchor" directive, OR have first-build stamp an initial anchor so commit-anchored drift detection works from session two onward.
  — [round1: defend] status=confirmed (distinct from C6): after reading `reconcile.py:128-134` and `drift.py:127-135`, A5 and C6 are TWO distinct gaps with one shared symptom. A5 is the *no-anchor* window: `_read_indexed_commit` returns None → `compute_reconcile_report` returns early (reconcile.py:129-134) so `drifted_features`/`unassigned` are structurally empty, and a freshly-built index has fresh mtimes so `rows` is empty too — the gate is blind regardless of any propagation fix. C6 is the *anchored* gap: the anchor IS set, the reconcile report DOES populate `drifted_features` (reconcile.py:174-179), but `compute_drift` never propagates it (drift.py:133-134,171-174 carry only unassigned+awaiting), so the gate's `_gate_relevant` (reconcile_gate.py:291-295) never sees committed modifications. A5's fix (stamp a first anchor at build) does NOT fix C6, and C6's fix (propagate `drifted_features`) does NOT fix A5 (no anchor → nothing to propagate). TWO seams.

- `dummyindex/context/reconcile_gate.py:315-317, 202-209` — **medium** — block-once is keyed on two independent escape hatches (`stop_hook_active` flag and the per-session memo) whose union means a single decline permanently silences the gate for the session even if much more drift lands afterward. — evidence: `decide_block` returns `None` on `stop_hook_active` (315) or `already_blocked` (317); `mark_blocked` (341) persists the memo after the first block. A long build-loop session that drifts code in turn 1, gets nudged, then drifts ten more features, is never re-gated. This is "strong prompt, not a trap" by design, but the memo is set *before* the agent has reconciled, so a dismissed-but-unreconciled session exits clean. — suggested fix: key the memo on "blocked AND reconcile-stamp advanced" (clear it when drift is still present at the next stop) so a dismissed gate re-arms once the index is still stale, rather than one-block-per-session regardless of outcome.
  — [round1: defend] status=confirmed: re-read `decide_block` (reconcile_gate.py:341) — `mark_blocked` fires immediately after the significance/source checks pass, BEFORE any reconcile happens. The memo records "we nudged once", not "the index is now clean", so a dismissed-but-unreconciled session exits clean on the next stop. This is a deliberate "strong prompt, not a trap" tradeoff (documented at module top, lines 15-18), so it is a *defensible pragmatic shortcut* for the trap-avoidance goal — but it does weaken the gate's guarantee. Confirmed as a real gap, classified pragmatic-by-design; medium stands.

- `dummyindex/context/reconcile_gate.py:241-270, 39` — **medium** — the "did this session drift source?" decision lives in the gate module and hard-codes its own `_NON_SOURCE_PREFIXES`, duplicating reconcile.py's tool-path definition; the two are kept in sync only by a test, not a shared seam. — evidence: line 39's tuple is commented "Kept in sync with reconcile.py's tool-path set ... test_gate_non_source_covers_reconcile_tool_paths locks the two together" — a test-enforced invariant standing in for a missing shared constant. `_session_drifted_source` (241-270) re-implements path attribution that `build/reconcile.py:_hidden` (165-172) already owns. — suggested fix: export one canonical non-source/tool-path predicate from `build/reconcile.py` and have the gate import it, so "what counts as source" is defined once.
  — [round1: defend] status=confirmed: re-confirmed the duplication — reconcile.py:359-360 defines `_TOOL_PATH_PREFIXES`/`_TOOL_PATH_EXACT` (`.claude/`, `.claude-design/`) and reconcile_gate.py:39 re-declares `_NON_SOURCE_PREFIXES` (`.context`, `.claude`, `.claude-design`) with a comment that a test locks them together. A test-enforced invariant is a code smell standing in for a shared constant. Real DRY/seam violation, but low severity — the test does catch divergence today. Confirmed; medium-to-low, leave at medium.

- `dummyindex/context/reconcile_gate.py:332-340` — **low** — the gate's proof-of-work check (`main_transcript`, `is_significant`, `_session_drifted_source`) couples the Stop-hook decision tightly to transcript-signal internals from `memory.transcript` / `memory.nudge`, so the reconcile-gate responsibility leans on the memory feature's seam to decide "was this substantive". — evidence: imports at 29-30 pull `is_significant` and `read_session_signal` from `domains/memory/*`; a change to the nudge's significance heuristic silently shifts when reconcile gating fires. — suggested fix: none required structurally (reuse is intentional and documented at module top), but note the cross-feature coupling so memory-heuristic changes are reviewed against gate behaviour.
  — [round1: concede] status=withdrawn: re-reading the imports (reconcile_gate.py:29-30) and the module docstring's explicit "Mirrors `memory/nudge.py`" framing, this reuse is deliberate and the predicate (`is_significant`) is a stable public seam of the memory feature, not an internal. The coupling is intentional layering, not a violation. Withdrawing — it was an observation, not a finding.

### Net architecture position after round 1

- Bug 1 (CLAUDE.md): **ONE seam** — extract `migrate_claude_md_location` into a `context/output/` domain helper with a corrected contract (fold ANY root CLAUDE.md, return a structured result), called by `runner.build_all` (covers init/ingest/install) and `refresh.py`. C1/C2/C4/A1/A2/A3/A4 and the test T7 all collapse onto this one extraction.
- Bug 2 (gate blindness): **TWO seams** — (C6) propagate `drifted_features` through `drift.DriftReport` → `_gate_relevant` → renderer; and (A5) close the no-anchor window by stamping a first anchor at first-build (or making the gate's anchor-less mtime branch authoritative with a "stamp your first anchor" directive). Neither fix subsumes the other. T13 is the missing test that locks both.
