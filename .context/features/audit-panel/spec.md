# Audit Panel and Onboarding Specification

confidence: INFERRED

## Intent

Provide deterministic Python contracts for two adjacent concerns: scaffold an adversarial-audit workspace that an agent skill can execute, and persist host-aware onboarding choices that audit and other council workflows consume. Core logic owns schema validation, legacy migration, default-plugin policy, model and mode resolution, persona roster resolution, and slug safety; CLI, installer, and atomic file-writing code are plumbing around those decisions.

## User-visible behavior

- `dummyindex context onboard` requires an existing `.context/` directory, validates flags into an immutable schema-v4 `Config`, writes `.context/config.json`, and echoes the exact stamped bytes written. The interactive question flow remains in the skill; this CLI is its persistence boundary (`dummyindex/cli/onboard.py:1-32`, `dummyindex/cli/onboard.py:111-179`).
- Non-default onboarding requires `--model`. It accepts repo, subdir, or explicit scope; light, standard, or deep mode; current or supported Claude model values; hook on/off; repeatable external-doc paths; and an optional host set of `claude`, `codex`, or `both` (`dummyindex/cli/onboard.py:7-22`, `dummyindex/cli/onboard.py:182-206`).
- `--defaults` chooses a host-aware baseline. Claude uses Sonnet, hooks, reviewed default-plugin declarations, and enabled default state; Codex uses the current session, no Claude hook, an empty ledger, and not-applicable `null`; both uses current, hooks, defaults, and enabled state (`dummyindex/context/domains/config.py:424-455`).
- Without an explicit `--platform`, defaults infer host coverage only from dummyindex's exact managed Claude and Codex guidance markers; no marker falls back to Claude. Explicit host/model/hook combinations that cannot be honored return exit 2 without writing (`dummyindex/cli/onboard.py:209-267`, `tests/context/domains/test_config.py:509-735`).
- Schema 4 requires `default_plugins_enabled` to be `true`, `false`, or `null`. `true` opts a Claude-enabled repo into reviewed defaults, `false` is a durable all-defaults opt-out, and `null` records a Codex-only not-applicable baseline (`dummyindex/context/domains/config.py:24-50`, `dummyindex/context/domains/config.py:160-212`, `dummyindex/context/domains/config.py:378-405`).
- Legacy reads normalize to schema 4 in memory. V1 `wire_superpowers` maps directly to enabled/disabled state; v2/v3 use a nonempty `wired` ledger as enabled and an empty ledger as disabled, except the exact canonical Codex baseline maps to `null` (`dummyindex/context/domains/config.py:214-301`, `dummyindex/context/domains/config.py:356-421`, `tests/context/domains/test_config.py:979-1100`).
- Installer migration rewrites only loadable stale schemas or renamed model values and preserves existing choices. Default-plugin reconciliation is a byte-identical no-op for Codex and explicit `false`; for Claude or both, it changes `null` to `true` and appends missing reviewed defaults after existing custom entries (`dummyindex/context/domains/config.py:525-564`, `dummyindex/context/domains/config.py:622-659`, `tests/context/domains/test_config.py:1103-1205`).
- Audit startup resolves model by flag then config and never silently invents one; it resolves mode by one shared depth seam: flag, audit-specific command depth, global mode, then standard (`dummyindex/context/domains/audit/workspace.py:104-140`, `dummyindex/context/domains/config.py:458-477`).
- An audit can scaffold without a pre-existing index. It validates or derives a safe slug, refuses an existing workspace unless forced, and atomically writes `audit.json`, `description.md`, `catalog.json`, plus `findings/` (`dummyindex/context/domains/audit/workspace.py:56-90`, `dummyindex/context/domains/audit/workspace.py:143-213`, `tests/context/domains/audit/test_audit_domain.py:215-279`).
- Persona cards come from the shipped markdown catalog. Roster resolution preserves an installed requested agent, otherwise selects the first capability match, otherwise uses `general-purpose`; when no roster source exists, shipped names remain untouched (`dummyindex/context/domains/audit/catalog.py:56-76`, `dummyindex/context/domains/audit/catalog.py:96-194`).

## Contracts

- `Config` is a frozen schema-v4 record with `scope`, `scope_path`, `mode`, `model`, `auto_refresh_hook`, external-doc and reconcile exclusions, per-command depths, the declarative `wired` ledger, tri-state `default_plugins_enabled`, writer version, and doc-guard settings. Subdir scope requires a nonempty path, and default-plugin state accepts only boolean or null (`dummyindex/context/domains/config.py:160-212`).
- `Config.from_dict(payload: dict[str, Any]) -> Config` strictly accepts schema versions 1 through 4, rejects bool-as-version and future versions, validates enums and collection shapes, performs legacy migrations, and always returns a schema-v4 object (`dummyindex/context/domains/config.py:214-301`).
- `default_config(*, platform: str = "claude") -> Config` accepts only `claude`, `codex`, or `both` and returns the corresponding host-aware baseline (`dummyindex/context/domains/config.py:424-455`).
- `read_config(context_dir: Path) -> Config | None` returns `None` only when absent and raises `ConfigError` on malformed or unsupported content; `write_config(context_dir: Path, config: Config) -> Path` stamps the current CLI version and performs a pretty, trailing-newline tmp-and-replace write (`dummyindex/context/domains/config.py:480-494`, `dummyindex/context/domains/config.py:662-682`).
- `migrate_config_in_place(context_dir: Path) -> bool` rewrites only a loadable stale schema or legacy model value and reports whether bytes moved (`dummyindex/context/domains/config.py:525-564`).
- `reconcile_default_plugins(context_dir: Path, *, platform: str) -> bool` preserves explicit opt-out, promotes not-applicable state only for Claude-enabled hosts, appends missing defaults after existing entries, avoids no-op writes, and raises on malformed config (`dummyindex/context/domains/config.py:622-659`).
- `resolve_depth(context_dir: Path, command: DepthCommand, depth_flag: str | None) -> CouncilMode` centralizes flag, per-command, global, and standard fallback precedence (`dummyindex/context/domains/config.py:458-477`).
- `onboard.run(args: list[str]) -> int` returns 0 after writing, 2 for invalid arguments or missing `.context/`, and emits the persisted config after its status line (`dummyindex/cli/onboard.py:111-179`).
- `AuditConfig`, `PersonaCard`, and `AuditStart` are frozen, JSON-shaped records. Audit deserialization requires schema 1 and an explicit model; `PersonaCard` retains the originally requested agent when roster resolution rewrites it (`dummyindex/context/domains/audit/models.py:22-126`).
- `resolve_model(context_dir: Path, model_flag: str | None) -> ModelChoice` uses flag then readable config and otherwise raises `ModelRequiredError`; `resolve_mode(context_dir: Path, mode_flag: str | None) -> CouncilMode` delegates to the shared audit depth resolver and translates config errors into audit errors (`dummyindex/context/domains/audit/workspace.py:104-140`).
- `ensure_audit(context_dir: Path, *, description, mode, model, scope=(), slug=None, force=False, personas_dir=None, roster=...) -> AuditStart` creates the deterministic audit workspace and resolved persona menu; it does not select or run the panel (`dummyindex/context/domains/audit/workspace.py:143-213`).
- `load_catalog(personas_dir: Path) -> tuple[PersonaCard, ...]`, `collect_roster(project_root: Path, context_dir: Path) -> tuple[RosterAgent, ...] | None`, and `resolve_catalog(cards, roster) -> tuple[PersonaCard, ...]` form the parse, observe, and pure-resolution pipeline (`dummyindex/context/domains/audit/catalog.py:65-194`).

## Examples

- `dummyindex context onboard --defaults --platform codex` persists `model: "current"`, disables Claude hooks, leaves `wired` empty, and records `default_plugins_enabled: null` (`dummyindex/context/domains/config.py:424-455`, `tests/context/domains/test_config.py:509-544`).
- A v3 canonical Codex baseline with no default-plugin field loads as `null`; the same schema with an explicitly empty non-Codex ledger loads as `false` (`tests/context/domains/test_config.py:1063-1085`).
- Reconciling that Codex baseline for `both` changes the state to `true` and appends all reviewed defaults in one run; reconciling with `platform="codex"` changes no bytes (`tests/context/domains/test_config.py:1103-1133`).
- Starting an audit for `Audit error handling in the CLI dispatcher` derives `audit-error-handling-in-the-cli-dispatcher`, writes the three structured/brief files and findings directory, and round-trips its explicit model and scope (`tests/context/domains/audit/test_audit_domain.py:215-247`).
- A shipped security persona resolves to an equipped security-capability agent while retaining `Security Engineer` as `requested_subagent_type`; a bare repo retains the shipped name because installed global agents are unknowable (`tests/context/domains/audit/test_audit_domain.py:519-587`).
