# Council critiques — source-docs

## critic-database (stage 3, data-integrity lens)

## Data integrity

- **Local `_atomic_write` diverges from the mandatory convention helper.** `writers.py:109-113` reimplements tmp+`replace` instead of routing through `write_text_atomic` (`domains/atomic_io.py:11-24`), so it inherits none of the convention's guarantees: no `cache/`-style Windows `PermissionError` copy-then-delete fallback (cf. `pipeline/io/cache.py:98-105`), and on Windows `tmp.replace(path)` can raise when a reader holds `INDEX.json`, leaving the `.tmp` orphan and the prior `INDEX` stale with no signal. Code wins over plan's "per the data-access convention" claim (`plan.md:85`) — the shape matches but the resilience does not.
- **Two-file write is not atomic as a unit.** `write_catalog` does two independent `_atomic_write` calls (`writers.py:21-22`); each file is individually atomic but the pair is not transactional. A crash or exception between line 21 and 22 leaves `INDEX.json` updated and `INDEX.md` stale (or vice-versa) — downstream consumers reading one as ground truth for the other (`output/docs.py` reads the catalog) can act on mismatched confidence. No cross-file integrity check or shared sha gate.
- **`mtime`-derived age signal is non-robust to clock skew / copied files.** `age_delta = newest_code_mtime - stat.st_mtime` (`catalog.py:158`) trusts filesystem `st_mtime` absolutely. A `git clone`/checkout/`cp`/rsync resets doc mtime to "now," making a genuinely stale doc look `fresh` and grade `high`; conversely a doc on a clock-skewed mount or restored-from-backup can show a future or ancient mtime and mis-bucket. The `fresh` floor at `-1s` (`catalog.py:27`) means even 1s of NTP jitter flips `recent`↔`fresh` — both map to `high` so harmless here, but the underlying signal has no skew tolerance and no content-based corroboration (sha256 is computed but never used for staleness).
- **Single scalar `newest_code_mtime` makes age coarse and brittle.** Age judges every doc against the one latest-touched file in the whole repo (`catalog.py:104,158`; plan.md:93 concedes this). One unrelated edit (e.g. a formatting sweep) refreshes `newest_code_mtime` for all docs, so a stale doc about an untouched subsystem still shows a small `age_delta` and can hold `high`. Determinism is preserved but the freshness verdict is only as good as that scalar.
- **`unknown` age → `high` is a deliberate mass-promotion with no stale-catalog backstop.** On a first build (or any run where `newest_code_mtime is None`), every low-broken-ratio doc grades `high` (`catalog.py:86-87,155-156`). Correct per the plan's first-build rationale (`plan.md:82`), but there is no separate "catalog itself is stale" detector — nothing flags that the catalog was built against a now-changed AST. Staleness detection is delegated entirely to the external cache manifest (`data-access.md:19-21`), which does not cover the source-docs `INDEX` files specifically.
- **`broken_ratio` float persisted unrounded in the entry; rounding only at the dict boundary.** `DocEntry.broken_ratio` stores raw `len(broken)/len(refs)` (`catalog.py:152,182`); the plan says `to_dict` rounds to 4 places (`plan.md:54`). The grading thresholds (`_classify_confidence`, `catalog.py:76,84`) compare the *unrounded* value against `0.10`/`0.40`, while the serialized JSON shows the rounded value — a doc graded off `ratio=0.10000001 > 0.10` can display `0.1000`, a small but real audit-trail divergence between the stored signal and the verdict it drove. Deterministic, but not self-consistent on inspection.
- **Confidence ordering relies on a lookup with a silent default.** Sort uses `DOC_CONFIDENCE_ORDER.get(e.confidence, 3)` (`catalog.py:191`); any confidence value not in the map silently sorts last rather than erroring. Today the set is closed (high/medium/low via `DocConfidence`), so byte-identical re-runs hold — but a future enum member would be absorbed without a contract failure, eroding the "council can trust it" determinism guarantee (`plan.md:79`).
- **JSON written with `sort_keys=False`.** `writers.py:21` serializes with `sort_keys=False`, unlike the convention's `meta.py:96` which adds `sort_keys=True` for byte-identical re-runs (`data-access.md:13`). Determinism here rests entirely on `to_dict` emitting keys in a stable insertion order; it holds for frozen-dataclass `to_dict`, but it is a weaker guarantee than the convention's explicit sort and would break silently if `to_dict` key order ever shifts.

## critic-security (stage 3, adversarial lens: arbitrary prose + `--docs` external roots)

- **Unbounded full-file read of arbitrary `--docs` content — memory-exhaustion DoS (High).** `catalog.py:138` `read_bytes()` + `refs.py:153` `read_text()` with no `st_size` pre-check (the `stat()` is taken at `catalog.py:139`; `size_bytes` already a field at `models.py:17`). `--docs DIR` with one multi-GB file → whole-file slurp + second `text` copy + `sha256(data)` over the buffer (`catalog.py:175`) OOM-kills the indexer. Fix: cap on `stat().st_size` before read; stream the hash.
- **Decompression-bomb DoS via office/PDF converters (High).** `.docx`/`.xlsx`/`.pdf` dispatched to `python-docx`/`openpyxl`/`pypdf` with no guard (`detect.py:107-110,123,162`). `.docx`/`.xlsx` are ZIP containers; `xlsx_to_markdown` accumulates every cell of every sheet into in-memory lists with no cap (`detect.py:166-180`). Blanket `except Exception` (`detect.py:115,154,185`) does not bound memory/time before detonation. Attack path: `--docs DIR` with one zip-bomb `.xlsx`. Fix: size-gate parsing, cap rows/cells/pages, consider subprocess + RLIMIT/timeout.
- **Symlinked doc files in a `--docs` root are read through to target — confinement bypass (Medium).** `walk_doc_files` uses `os.walk(root)` default `followlinks=False` (`common.py:34`) yet still yields symlinked *files* and `p.resolve()`s them (`common.py:42`); bytes then read at `catalog.py:138`/`refs.py:153`. `notes.md -> /etc/passwd` in a writable `--docs` tree exfiltrates the target's bytes (`sha256`, `size_bytes`, headings, backticked tokens) into `INDEX.json` with the resolved absolute path. No symlink check at all, unlike the deep walker (`detect.py:352,414-421`). Fix: skip `p.is_symlink()` or assert resolved target stays under the root (also `discovery.py:60-62`).
- **External `--docs` bytes carry in-repo authority — content-trust (Medium).** `is_external` (`catalog.py:91-95`) is display-only; external docs feed the same grader and `generate_project_md` can lift an external README's first paragraph into `PROJECT.md` (`output/docs.py:122-194`). A planted sibling-dir doc can become the project's self-description. Fix: never source `PROJECT.md` from `is_external` docs; tier external docs below in-repo in confidence.
- **`_attribute_source_root` compares resolved doc path vs unresolved `repo_root` (Low).** `catalog.py:216` re-resolves only `extra_doc_roots` (`:211`), not `repo_root`; sound today because the caller resolves at `:124`, but a future unresolved/symlinked `repo_root` mis-attributes in-repo docs as external. Implicit contract; enforce at the helper boundary.
- **Doc content trusted as signal — bounded, informational.** Harvested backticked tokens (`refs.py:191-205`) + JSON keys (`keys.py:13-47`, capped `limit=5000`) only widen the known-name set; a hostile doc can at most *suppress its own* `broken_refs` to self-grade `high`. The council consumes `confidence` as evidence, so a doc can lie its way to `high`; `broken_count ≥ 4` floor + ratio thresholds (`catalog.py:76-87`) bound the blast radius. No code-exec, no cross-doc tampering.
- **Non-finding (regex): no catastrophic backtracking.** `_INLINE_CODE_RE`/`_CODE_FENCE_RE`(DOTALL)/shape regexes (`refs.py:13-21`) measured against adversarial input — per-token anchored ≤160 chars (`refs.py:13,96`); whole-doc passes ran linearly (~0.03s/3 MB). Recorded so it is not re-raised. Real doc-DoS is unbounded I/O + converters, not regex.

## Stage 3 — Product-surface critic (critic-product)

Lens: a user relying on the doc catalog. Identifiers verified in `.context/map/symbols.json` before quoting plan/code.

- **No docs at all — clean self-recovery hint, but closed extension set is silently invisible.** Zero-doc build emits `_No documents discovered. Pass --docs PATH..._` (`writers.py:46-51`), but `.adoc`/`.org`/`.tex`/`.ipynb` files in a `docs/` dir are simply not in `_DOC_EXTENSIONS` (`discovery.py:27-32`) and are never reported as skipped — a user with org-mode docs gets a confusingly empty catalog with no hint why.
- **Many docs — uncapped `INDEX.md` table vs invisibly-capped `PROJECT.md` (`[:5]`, no "+N more").** `_render_catalog_md` renders every doc (`writers.py:56-62`) while the curated `PROJECT.md` notable list truncates to `[:5]` (`docs.py:274`) with no "+N more" affordance — unlike the low-conf broken-ref list which does show `_… +{more} more_` (`writers.py:85-86`). The `PROJECT.md` cap is invisible.
- **Grading surprise: `unknown` age + clean → `high`, rendered identically to genuinely-fresh `high`.** First-build docs grade `high` purely on broken-ratio (`catalog.py:86-87`) with age column `unknown` (`writers.py:61`); the bolded confidence verb dominates the eye over the age caveat. No visual tier between "high (fresh)" and "high (freshness-unknown)".
- **Grading surprise: a ZERO-ref doc grades on age alone — prose-only README can be `high` while saying nothing checkable.** `broken_ratio = 0.0` when refs empty (`catalog.py:152`) ⇒ `high` if recent/unknown (`catalog.py:84-87`); broken column shows `—` (`writers.py:57`) but the `high` badge implies "safe to quote" (`docs.py:269`) when nothing was cross-checked. `referenced_count == 0` should tier separately.
- **Broken-ref false-positive: single global name-set, no per-doc scoping.** `find_broken_refs` matches against `symbol_names ∪ file_paths ∪ extra_names ∪ whitelist` with no scoping (`catalog.py:146-151`); a monorepo `--docs ../other-svc/README.md` citing a real-but-other-repo symbol is flagged `broken` and dragged to `low` (`writers.py:65-93`), labelled "no longer in the AST" — a false signal, remediation not surfaced.
- **Staleness clarity: human day-count only renders for `stale`/`old`; `aging` docs get no number.** `writers.py:88-92` gates the "X day(s) before the newest code change" line on `stale`/`old`; an `aging` doc (31–90d, `catalog.py:29`) shows only the bucket word. Most actionable number hidden for the middle bucket; and that number is computed against the global `newest_code_mtime` so it can read alarmingly large from one unrelated edit.
- **Staleness clarity: catalog carries no "I am stale" banner.** `generated_at` exists (`catalog.py:195`) but is never compared to current source; `INDEX.md`'s `_ADVISORY_BANNER` (`writers.py:31`) is generic, not freshness-aware. The drift hook covers feature docs, not the source-docs `INDEX` (prior Data-integrity §5). A reader cannot tell whether verdicts were computed against today's code.
- **`--docs` UX: missing path warns-and-skips to stderr, run exits 0 — a typo'd root yields a confidently-smaller catalog.** `resolve_doc_paths` (`common.py:173-178`) drops the path and continues; in CI the warning scrolls past and the user believes their docs were ingested. No "ingested N of M roots" summary, no strict mode.
- **`--docs` UX: external docs sort INTERLEAVED with in-repo by confidence — an external `high` outranks an in-repo `medium`.** `is_external` is signalled only by an absolute path/non-clickable link (`writers.py:104-106`); the confidence sort (`catalog.py:191`) actively elevates less-trusted-provenance docs and the table has no "external" column.
- **`--docs` overlap: nested/symlinked roots double-attribute by CLI order.** `_attribute_source_root` returns the first matching `extra_doc_roots` entry (`catalog.py:209-214`); `--docs /a --docs /a/sub` attributes `/a/sub` docs to `/a`, with no user-facing precedence rule.

### Cross-review notes (Security-flagged interplay)

- **Security §16/§17 has a product consequence: no size column / skip-report, so the OOM-causing doc is invisible.** `INDEX.md` shows no `size_bytes` column (model field, `models.py:17`); when the indexer dies on a huge `--docs` file the user gets no row pointing at it. The `_MAX_DOC_BYTES` cap should also emit a visible "skipped (too large)" row.
- **Security §18: symlinked doc files are invisible at the surface — resolved target lands in `path` with no "via symlink" marker.** `notes.md -> /etc/passwd` renders the resolved `/etc/passwd` as a normal catalog entry (`catalog.py:164`); beyond skipping, the surface should mark/omit symlinked entries.
- **Security §19 partially already mitigated in code.** `_description_from_catalog` already excludes externals — `not d.is_external and d.confidence != DocConfidence.LOW` for READMEs, `not d.is_external and d.confidence == DocConfidence.HIGH` for the title fallback (`output/docs.py:206,213`). The `PROJECT.md` *description* path is already confined to in-repo docs; code wins over the §19 "can become the project's self-description" claim for the description specifically. Residual §19 risk (external tokens widening the trusted name-set) stands — recommend narrowing §19.
