# Council critiques — session-memory

## critic-database (data integrity)

## Data integrity

- **State-file read-modify-write is racy under concurrent hooks.** `mark_nudged` (`nudge.py:56-65`) and `mark_blocked` (`reconcile_gate.py:212-226`) both do load→mutate-dict→prune-to-100→`write_text_atomic`. The `tmp.replace` is atomic (`atomic_io.py:21-24`), but the load-then-write window is unlocked: if a Stop hook and a SessionStart/PreCompact path write `nudge-state.json` / `reconcile-gate-state.json` concurrently, the later writer clobbers the earlier writer's whole dict — a freshly-recorded `session_id` memo can be lost, re-arming a one-shot nudge/block (double-fire). No file lock or atomic merge guards the sequence.
- **Pruning to 100 is mtime-blind and can evict a live session's own memo.** Both prunes keep the 100 highest `nudged_at`/`blocked_at` ISO strings (`nudge.py:62-64`, `reconcile_gate.py:219-225`). Entries written with an empty/UTC-vs-local-mixed timestamp sort to the bottom; once >100 sessions accumulate, a just-written memo with a low-sorting timestamp can be dropped in the same call, defeating block-once. Keying retention on recency-of-write rather than the embedded string would be safer.
- **Roll idempotence depends on `_sort_desc` being a fixpoint; re-sort on no-move is skipped but partial writes aren't all-or-nothing.** `roll_tiers` correctly early-returns byte-identical on no-move (`roll.py:80-81`), but when a move occurs it issues up to three independent `write_text_atomic` calls (`roll.py:83-96`) with no cross-file transaction. A crash/interrupt between the `now.md` write and the `recent.md` write leaves a section deleted from `now` but not yet appended to `recent` — loss, not dup. Re-running `roll_tiers` does not recover it (the section is already gone from `now`). The cascade is per-file atomic but not multi-file atomic.
- **Roll can silently drop sections whose date is exactly the boundary or unparseable mid-format.** Partition predicates compare `section_date(heading)` ISO strings/ordinals (`roll.py:67-73`); a heading whose date parses on one tier's key path but not another, or an out-of-range ISO (`_ordinal` calls `date(...)` at `roll.py:38-40` which raises `ValueError` on e.g. month 13), would propagate an exception out of an otherwise emit-only path. `roll` is not wrapped in the hooks' never-fail guard at the partition step — a malformed agent-authored heading could abort the roll rather than pin-in-place.
- **Transcript reader silently undercounts on partial/duplicate transcripts — threshold flap.** `read_session_signal` swallows `OSError` and bad JSON lines (`transcript.py:91-115`) and, by Decision 2 (`transcript.py:6-8`, confidence high, identifier confirmed in symbols.json), does **no** cross-transcript dedup. A multi-transcript session right at `LONG_OUTPUT_TOKENS=40_000` (`nudge.py:23`) or the gate threshold can over- or under-count `output_tokens`, making the one-shot nudge/block non-deterministic across runs of the same session. Acceptable as a heuristic per the plan, but it is a correctness caveat for any consumer treating the signal as authoritative.
- **`find_main_transcript` mtime fallback can bind to the wrong session.** With no `CLAUDE_CODE_SESSION_ID`, it picks the newest `*.jsonl` for the cwd slug (`transcript.py:72-76`); concurrent sessions in the same project would read each other's transcript, attributing one session's `edited_paths`/tokens to another — wrong source-drift attribution feeding the gate. Session-id-keyed lookup is correct; the fallback is the integrity hole.
- **Manifest cross-filter trusts a gitignored, per-machine cache as truth-source.** The three-oracle filter reads `cache/manifest.json` sha256 to kill mtime false-positives (`drift.py:144-175,338-352`; data-access.md:19-21). A stale or absent manifest (fresh clone, cleared cache) collapses the authoritative sha oracle, so the gate/report fall back to raw mtime — correct-but-noisier, and the failure is silent (no signal that the sha oracle was unavailable).

## critic-security (adversarial)

## Security

- **CRITICAL — `transcript_path` from hook stdin is an unbounded arbitrary-file-read primitive.** `_resolve_transcript` returns `Path(tp)` from untrusted Stop/PreCompact stdin with no containment check (`memory.py:48-50`); it flows into `read_session_signal` (`nudge.py:122`, `reconcile_gate.py:334`, `breadcrumb.py:105`) which `open()`s and streams it (`transcript.py:92`). A crafted `{"transcript_path":"/etc/passwd"}` reads any file and pipes it into the gate decision and the model context. Fix: require `tp` resolve under `_projects_root()`.
- **CRITICAL — transcript-derived strings reach the model as directives unescaped (prompt injection).** `_edited_paths_in` takes `input.file_path` verbatim (`transcript.py:138-140`); the resulting drifted-feature / `unassigned_new_files` strings are joined into the Stop block `reason` the agent must act on (`reconcile_gate.py:106-112`, `render_block`) and into the breadcrumb `Touched:` body (`breadcrumb.py:31`) re-emitted by `render_session_start` (`emit.py:47-49`). A `file_path` of `"x. IGNORE PRIOR INSTRUCTIONS…"` becomes a next-session instruction. Fix: treat transcript text as data — escape/length-bound before any model-facing string.
- **HIGH — markdown/heading injection corrupts the tier store and date logic.** Unescaped path text written to `now.md` (`breadcrumb.py:34-39`); a newline+`## 2099-01-01` forges a heading (`split_sections` keys on `## `, `parse.py:34`; `section_date` regex `parse.py:50-53`), making `real_handoff_saved_today` suppress the nudge forever (`nudge.py:77`) or driving `roll_tiers` to evict real sections (`roll.py:67-75`). Fix: strip `\n`/leading-`## ` from `changed_files`.
- **HIGH — DoS via a single oversized JSONL line.** `for raw in fh:` (`transcript.py:93`) has no byte/line cap; one newline-free multi-GB line forces a full read + `json.loads`, OOM-killing every session-end on the attacker-controlled path (item 1). `except OSError` (`transcript.py:114`) won't catch `MemoryError`. Fix: windowed/bounded read + nesting cap.
- **MEDIUM — mtime/slug discovery reads a different session's transcript.** No-session-id fallback picks newest slug-dir `*.jsonl` (`transcript.py:72-76`); `_encode_slug` is non-injective (`transcript.py:54-55`), so `/a/b` and `/a-b` collide — a sibling session's transcript is attributed here. Fix: scope by exact resolved cwd, prefer env session id.
- **MEDIUM — `roll`/`_ordinal` raise on a forged out-of-range heading.** `date(year, month, day)` (`roll.py:38-40`) raises `ValueError` on `2099-13-40` injected via the heading-injection path; no guard at the partition step, and the same store feeds the Stop-hook nudge. Fix: try/except → treat unparseable as undated (pin-in-place).
- **LOW — captured secret-bearing paths surfaced in stdout directives.** Block `reason` / nudge `additionalContext` echo paths (`reconcile_gate.py:124`, `nudge.py:91-98`); a path read via item 1 leaks into the directive. Covered by sanitizing item 2.


## Product surface critique (critic-product, stage 3)

- Promotion to `core-memories.md` — NO deterministic verb promotes facts to core; the CLI only *reads* core whole at SessionStart (`emit.py:41,50-51`), and "promote durable facts" lives only in skill prose (`dummyindex-remember/SKILL.md:38-39`). Plan calls core "durable promoted facts … agent-curated" (`plan.md:121-122`) but ships no `promote` mechanic — gap: edit/save without the remember skill and core stays empty forever; nothing in the HANDOFF line (`emit.py:53-56`) tells the user core exists or how to fill it.
- Roll wall-clock fragility — `now→recent` keys on `iso < today_str` from local `date.today()` (`roll.py:54,67-69`); a handoff written before midnight and rolled after (or a UTC CI runner) silently relocates a section the user still treats as "now." Gap: tier membership is date-string fragile, not session-scoped; no "still current" pin.
- No-memory / empty store — clean: `render_session_start` returns `None` on absent dir or all-empty tiers (`emit.py:36-43`); `ensure_memory_store` idempotent/non-destructive (`store.py:15-29`). Gap: `archive.md` is never emitted (only now/recent/core read at `emit.py:39-41`) — once a section rolls past `recent_keep_days=7` it is invisible to every future session. Four tiers stored, three surfaced; archive is write-only from runtime view.
- Corrupt memory — a malformed heading date raises `ValueError` out of `_ordinal` (`roll.py:38-40`) and aborts `memory roll`; only feedback is a traceback, no "bad heading, skipping" message, no validation/repair. Store is hand-editable markdown (`plan.md:139-141`) yet one mistyped date breaks the roll with no graceful degradation.
- Suppression-when-remember-present — detection is just "does `<root>/.remember/` exist" (`detect.py:7-14`); when present emit/nudge/breadcrumb all silence (`emit.py:33`, `nudge.py:113`, `breadcrumb.py:125`). Gaps: (1) a stale leftover `.remember/` permanently silences memory with zero signal WHY the block vanished; (2) all-or-nothing — an existing dummyindex store stops being emitted with no migration/merge when `remember` is installed; (3) asymmetry — `memory roll` (`cli/memory.py:124-131`) is NOT gated by `remember_plugin_present`, so it keeps rewriting `now.md` while the plugin owns injection — mutating a store nothing reads.
- Capability hinted, not delivered — `recent_keep_days=7` (`roll.py:47`) and `LONG_OUTPUT_TOKENS=40_000` (`nudge.py:23`) are hardcoded; plan Open question (`plan.md:169-172`) calls config "open" though the gate already reads `config.json` for `auto_council`. A user wanting a quieter nudge or longer recent window must patch source.
- Nudge UX — fires once per significant unsaved session (`nudge.py:101-129`), correctly suppressed after a same-day real handoff (`nudge.py:68-77`). Gap: significance is `subagent_file_count > 0 OR tokens >= 40_000` (`nudge.py:26-30`), so a single trivial read-only sub-agent trips it — heavy-delegation workflows get nudged almost every session with no opt-out short of installing `remember` or editing the constant.

### Cross-review note (UX/operator impact of Security CRITICALs)

- The arbitrary-read + unescaped-re-emit CRITICALs compound the surface gaps: the silent stand-down means a user can't distinguish "suppressed because remember present" from "block poisoned/empty," so an injected forged future-dated heading that flips `real_handoff_saved_today` true and kills the nudge is operationally invisible. With archive write-only and core un-promotable, the auto-emitted `now/recent` block is the user's only durable model-facing memory — exactly the prompt-injection channel. The "never fail, exit 0" resilience (`plan.md:88-91`) means a corrupted/attacker-shaped store fails open and silent: no error, no diff, no signal. Any sanitization fix should add a user-visible signal when a tier is rejected/truncated so suppression is never indistinguishable from compromise.
