## Data integrity

- **Atomicity holds, but via a local clone, not the canonical helper.** `_atomic_write` (`dummyindex/context/output/instructions.py:530-534`) does `mkdir(parents=True)` → write `path+".tmp"` → `tmp.replace(path)`, so a concurrent reader (statusline badge, drift hook) never sees a half-written doc — the one mandatory rule (`conventions/data-access.md:5-7`) is satisfied. The concern is *duplication*, not correctness: it byte-for-byte reimplements `write_text_atomic` (`domains/atomic_io.py:11-24`) instead of calling it, exactly the "don't bypass `write_text_atomic` with a plain reimplementation" pitfall (`conventions/data-access.md:27`). Verified equivalent (same `encoding="utf-8"`, no normalization), so no data hazard today — fix path is dedup: route the three `write_*` (`:505,:509,:526`) through `write_text_atomic` and delete `_atomic_write`. Low severity.
- **No Windows `PermissionError` fallback.** Unlike the cache writer's copy-then-delete guard (`pipeline/io/cache.py:98-105`), `_atomic_write`'s `tmp.replace(path)` (`:534`) can raise on Windows if a reader holds the target open. For these append-rarely, regenerated-wholesale docs the window is tiny and `build_all` is the sole writer, so impact is low — but it's a second reason the local copy is strictly worse than the shared helper, which is the natural place to add the guard once.
- **Idempotent / deterministic regeneration confirmed.** Output is fully content-determined: `generate_*` are pure string-returning (no disk read, no clock, no randomness), the static docs are frozen literals (`_HOW_TO_USE` `:24-109`, `_PLAYBOOK_BODIES` `:338-487`), and emission order is stable — `PLAYBOOK_IDS = tuple(sorted(_PLAYBOOK_BODIES))` (`:490`) and `_select_architecture_docs` sorts by `DOC_CONFIDENCE_ORDER` (`:316-332`). Re-running `build_all` over an unchanged index produces byte-identical files; `replace()` makes the swap atomic. No partial-write or duplicate-emission risk.
- **`KeyError`-on-miss is the right failure mode, not silent corruption.** `generate_playbook_md` rejects an unknown id with a `KeyError` listing valid ids (`:493-499`) rather than emitting an empty/placeholder doc — a regeneration over a bad id fails loud, never writes garbage. Good.

## Security

Adversarial lens: index-derived strings (dir names, file paths, source-doc titles) flow unsanitized into `architecture/overview.md`, a doc future agents read as authoritative navigation. Trust boundary = repo filesystem + committed `.md` files; output isn't browser-rendered, so the payload is markdown/prompt injection into an agent-facing doc, not XSS. 5 threats:

- **HIGH — source-doc title rendered verbatim.** `title = f" — {d.title}"` → `f"...(**{d.confidence}**{title})"` (`instructions.py:261-265`), `d.title` extracted from doc H1 (`domains/source_docs/catalog.py:144`, `domains/source_docs/models.py:14`). A `docs/architecture.md` with an instruction-shaped H1 is auto-selected by the `"architecture"` filename signal (`:306-313`) and high-confidence-sorted to the top (`:331`) of the "Documented architecture" list. Escape/strip control + markdown tokens, collapse to one line, cap length.
- **HIGH — markdown-link injection via `d.path`.** `f"- [`{d.path}`](../../{d.path}) ..."` (`:263`) uses `d.path` unescaped as both link text and href; filesystem-legal `)`/`](`/`../` in a path forge a trusted-looking target or break the link. Percent-encode href, escape link text, reject `..`/metachars.
- **MEDIUM — dir/file names unescaped in table + list.** `` `{s.name}/` `` in a 5-col table row (`:232`) and `` `{f.path}` `` in root list (`:244`); a POSIX-legal backtick/pipe/newline in a name corrupts the table or injects a row. Backtick/pipe-escape cells; reject newlines.
- **LOW — no bound on catalog/layout size.** `_select_architecture_docs` + loops emit one line per match unbounded (`:228-266`); doc-flooding buries the AST ground truth and burns agent context. Cap + elide.
- **NOTE — atomic write is not a content-trust control; no secrets emitted.** `_atomic_write` (`:530-534`) only prevents torn reads; the only sensitive property is integrity of agent-facing instructions (the injection findings). `HOW_TO_USE`'s detect-secrets guidance (`:87-102`) is recommended template text, not a control this feature enforces.
