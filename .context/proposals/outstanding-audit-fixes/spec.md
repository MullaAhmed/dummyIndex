# Spec — Implement outstanding audit + council findings

> Cache-poisoning hardening, reality-check path confinement, clustering determinism, atomic confidence mirror, query UX, and deferred perf cleanups.
> **Revised twice** — after a 3-critic panel (reuse/arch, risk, testability), then after the `/dummyindex-audit` meta-audit (`audits/audit-plan-and-results/report.md`) — see `## Revision notes` at the end for what was folded.

## Intent

The `/dummyindex-audit` panel and the deep recouncil of `extraction-pipeline`, `reality-check`, and `cli-dispatch` surfaced **confirmed, code-grounded** concerns that were documented but not yet fixed. Four themes:

1. **Trust-boundary leaks** (highest severity) — the content-addressed cache and the reality-check verifier read filesystem paths driven by untrusted/LLM-authored input with no containment: cache poisoning, out-of-tree reads, and a `--feature` write primitive.
2. **Committed-artifact determinism** — `symbol-graph.json`'s `community` field differs across machines by clustering backend (single-backend determinism already shipped in `c574d41`; the **cross-backend** gap remains — the GATE); `hyperedges` are emitted unsorted.
3. **Cross-file write atomicity** — reality-check's two-file confidence mirror is not atomic-together and omits `sort_keys`, and `--demote` silently mutates confidence with no report signal.
4. **Deferred cleanups** — files read up to 3× per build (P2), `latest_status` duplicated (M5), `query` arg-handling surprises, and an in-place mutation of cached dicts that violates the immutability convention.

The audience is the maintainer and any agent navigating `.context/`: the index must stay byte-reproducible, the verifier must not be a filesystem read/write oracle, and the cache must not silently serve attacker-shaped graph data. Fixes **reuse** the existing atomic-write helper and a new shared path-confinement primitive rather than re-implement per call site.

## Contracts

**New shared primitive** — `dummyindex/pipeline/io/paths.py` (lowest layer; the legal `context → pipeline.io` arrow is already used, e.g. `context/__init__.py` imports `pipeline.io.is_git_repo`). It **consolidates the ~6 existing inline `x.resolve().relative_to(root)` guards** (`cli/common.py:41-45`, `context/reconcile_gate.py:64-68`, `context/build/incremental.py:108-112`, `context/build/manifest.py:105`, `cli/check.py:78`, `cli/equip/discover.py:257`) into named, tested symbols — net-new *symbol*, not net-new *logic*. Re-exported from `pipeline/io/__init__.py` (the package's test surface).
- `resolve_under_root(candidate: Path, root: Path) -> Path | None` — **pure** (only `resolve()`): returns the resolved path iff it is `root` or a descendant of `root.resolve()`; else `None`. Catches `../` escapes *and* absolute-path joins (`root / "/etc/x"` == `/etc/x` resolves outside). Must accept **both already-resolved and not-yet-resolved** candidates (2 of the 6 guards above — `cli/common.py:42`, `reconcile_gate.py:66` — pass already-resolved paths).
- `is_safe_read_target(path: Path, *, max_bytes: int) -> bool` — performs `os.lstat`/`os.stat` (this helper **does** touch the filesystem): rejects symlinks (`S_ISLNK`), non-regular files (FIFO/device), and files larger than `max_bytes`. Returns a bool; never raises (no typed-exception needed).

**Reused symbols (do not re-implement):**
- `write_text_atomic` (`dummyindex/context/domains/atomic_io.py:11`) — the canonical tmp+`replace` writer. **Wave 1 retires `reality_check/render.py:_atomic_write`** — used at **three** sites (in-file `write_report` at `render.py:16-17`, the import at `confidence.py:16`, and the **re-export at `reality_check/__init__.py:74`**); repoint the first two at `write_text_atomic`, drop the re-export, then delete the def, so the later mirror/report tasks fan out against one stable writer.
- `cache_dir_override` (`context/build/common.py:130`) — the in-repo cache-dir setter, which today sets the **env var** `DUMMYINDEX_CACHE_DIR` (`:134`). Because `cache_dir()` reads that env var with **precedence over** the threaded `root` (`cache.py:51-55`), the cache confinement must **re-route `cache_dir_override` off the raw env var** onto a trusted in-process channel, then confine the ambient env-var value (see Cache trusted-root below).
- `usage_error` (`cli/common.py:47`) — the shared CLI error emitter `query` must route through.
- `is_git_repo` / `resolve_git_dir` (`pipeline/io/git.py`) — the anchor for a trustworthy `repo_root` in reality-check.

**The "trusted root" per call site (the panel's highest-leverage gap — name it before implementing):**
- **Cache** — the threaded `cache_root` (`extract/__init__.py:163,169`) is **discarded whenever `DUMMYINDEX_CACHE_DIR` is set** (`cache.py:51-55` returns in the `if override:` branch first), and `cache_dir_override` sets exactly that env var — so the original "the trusted `cache_root` channel keeps the override working" framing was **inverted**. Corrected model: `cache_dir_override` must set a **trusted in-process** cache dir (not the env var), and `cache_dir()` confines only the **ambient** `DUMMYINDEX_CACHE_DIR`, silently falling back to `<root>/.context/cache/` on escape (never raising — `test_cache_env_var_is_restored`). Caveat: `DUMMYINDEX_CACHE_DIR` is a **documented user-facing opt-out** (`cache.py:47-49`, `CHANGELOG.md:1095`) — either preserve it by distinguishing the trusted override write from an ambient value, or deprecate it as an intentional breaking change and update the docstring + CHANGELOG.
- **Reality-check** — the trusted root is the **resolved** git toplevel (via `is_git_repo`/`resolve_git_dir`) or the CLI-resolved `--root`, **not** "any ancestor of `context_dir`" (which admits `/`).
- **CLI** — `--root` as already resolved by `resolve_context_root`.

**Invariants the fixes must preserve:**
- `symbol-graph.json`, `map/*.json` stay byte-identical across runs **and** machines (`conventions/data-access.md`). Note the **hyperedge sort (Task 5)** and the **GATE `community` decision (Task 6)** cause a **one-time** committed-bytes change on the next rebuild — expected, not a regression.
- Reality-check behaviour on *legitimate* in-repo citations is unchanged (no new false negatives) — golden test over this repo's own docs **plus** a **nested-`.context/`** fixture (a subtree layout where `context_dir.parent` is not the repo root).
- Within one `build_all`, each source file's content is read once and **all passes see the same bytes** (P2 closes a latent stale-node/new-byte hole on the cache-hit path).
- No public import path breaks; all changes immutable-by-default (`conventions/coding-practices.md`); the cache-dir change must not mass-invalidate existing entries (minimal accept-superset schema).

**Seams touched:** `pipeline/io/{paths.py(new),cache.py,git.py}`, `pipeline/io/__init__.py`, `pipeline/extract/__init__.py`, `pipeline/build/references.py`, `pipeline/build/structure.py`, `pipeline/extract/generic.py`, `context/build/incremental.py`, `context/build/common.py`, `analysis/cluster.py`, `export/graph.py`, `context/domains/reality_check/{verify,confidence,render,__init__}.py`, `cli/reality_check.py`, `cli/query.py`, `cli/help.py`, `context/domains/{audit/log.py,council.py,log_scan.py(new)}`.

## Acceptance

**Shared primitive**
- [ ] `resolve_under_root` / `is_safe_read_target` exist in `pipeline/io/paths.py`, re-exported from `pipeline/io/__init__.py`, with unit tests covering: in-root path, `../` escape, **absolute-path** escape, symlink target, oversize (> `max_bytes`), FIFO/non-regular, **and an already-resolved in-root candidate**. A legitimate in-root path returns the resolved path (positive control).

**Cache hardening (extraction-pipeline)**
- [ ] An **ambient** `DUMMYINDEX_CACHE_DIR` (set in the environment, **not** via `cache_dir_override`) pointing outside the repo root is ignored → **silently** falls back to `<root>/.context/cache/` (no raise — `test_cache_env_var_is_restored` still passes); and `cache_dir_override` (re-routed off the env var onto the trusted in-process channel) still directs the cache to its target — both proven by tests. The chosen policy for the documented `DUMMYINDEX_CACHE_DIR` out-of-repo opt-out (preserved-but-trusted vs deprecated-with-doc-update) is recorded.
- [ ] `load_cached` returns a miss (re-extract, never merge) for: a non-dict payload, `nodes`/`edges` not list-typed, or a **node** lacking a string `id` (edges are not required to have `id`); **a valid entry produced by the current `save_cached` still hits** (no mass invalidation) — proven by tests feeding the three malformed shapes + one real entry produced *through `save_cached`*.
- [ ] `_body_content` on a `.md` with YAML frontmatter + a later in-body `---` returns the full body below the **closing fence** (line-anchored `^---\s*$`), and a **non-bare `---hack`** in the body does **not** truncate it (the sharper bug); a frontmatter-without-closing-fence file still hashes whole-file (no regression) — proven by unit tests on `_body_content`.

**Extraction walk (extraction-pipeline)**
- [ ] An extraction walk with `follow_symlinks=True` over a repo containing a symlink whose realpath is outside the root does **not** read the out-of-tree target — proven by a test asserting no node/edge carries out-of-root content (containment filtered at **`collect_files` leaf emission**, the single in-scope chokepoint before both the cache-hash read and the extractor read). This is **walk-time** — a post-enumeration symlink swap is a documented residual.
- [ ] When two distinct symbols normalize to the same `global_label_to_nid` key, a `calls`/`uses` edge to that label is disambiguated or skipped (never silently bound to the last-iterated node) — proven by a test with two same-named symbols in different files.
- [ ] The `id_remap` pass builds new dicts (`{**n, "id": …}`) instead of mutating the cache-returned dicts in place — proven by a test that **patches `load_cached` to return a captured sentinel dict** and asserts that object's `id` is untouched after `extract` (a fresh re-`load_cached` returns a new copy and would false-pass, since the aliasing is on the object from the *first* call).

**Determinism (extraction-pipeline)**
- [ ] `hyperedges` are sorted by a **total** key (e.g. `json.dumps(h, sort_keys=True)` — a single field is insufficient given the open hyperedge schema) before dump; two consecutive rebuilds emit byte-identical `hyperedges` — proven by a test that exercises a **non-empty** `hyperedges` list.
- [ ] **GATE-dependent** (see Open questions; reword per chosen strategy, do not tick until decided):
  - strategy (a) require graspologic → build fails fast with the install hint when the backend is absent (monkeypatch the import to raise; assert exit/raise). **Env-robust** (graspologic is absent in the test venv, so the fail-fast half tests as-is); the Leiden-present happy path needs graspologic installed in CI;
  - strategy (b) always-Louvain-for-committed → `community` is byte-identical across two builds. **A "spy asserts Leiden never called" assertion is VACUOUS when graspologic is absent** (the import already raises and falls to Louvain) — if (b) is chosen, the spy must run with graspologic **installed + `leiden` monkeypatched**, and CI must pin that backend;
  - strategy (c) exclude `community` → **breaking (see Open questions): feature-grouping re-reads the committed `community` (`runner.py:182-187`→`builder.py:76`), so (c) also requires re-routing `builder` to an in-memory partition** before the key can be dropped; the key is then absent from committed bytes and on-demand computation is tested separately.
  - The chosen single-backend test should use a graph large enough to hit the recursive `_split_community` second partition (`cluster.py:113-128`).

**Reality-check trust boundary**
- [ ] A citation resolving outside `repo_root` yields `ambiguous`/`contradicted` and **never** opens the target — proven by spying `pathlib.Path.open` (the real read call site, `verify.py:144`) and asserting it is never invoked for an out-of-root path; **every** return branch of `_resolve_cited_path` is gated. **The proof strings MUST be extension-bearing** (`` `../../secrets.env:1` ``, an **absolute** `/tmp/<name>.json:1`, or an in-repo symlink escaping root) — `_FILE_LINE_RE` (`extract.py:32-34`) requires a `.ext` before `:line`, so extensionless paths like `/etc/passwd:1` are **grammar-rejected before reaching the verifier** and would make the test a **false pass** against unpatched code; keep one extensionless string as a control explicitly labelled "rejected by the citation grammar, not by confinement". The chosen in-repo-symlink policy is documented.
- [ ] `reality-check --feature ../../escape` is rejected at the **CLI boundary** (`cli/reality_check.py` right after parsing `--feature`) with a non-zero exit and **no write** outside `.context/features/` — proven by a test; the same guard also protects the verifier path.
- [ ] A `meta.json` `root` that is **not** the git toplevel / resolved root (e.g. `/`) falls back to the trusted anchor; a `root` that **is** the genuine root is honored unchanged (positive control) — both proven by tests, including a **new nested-`.context/`** fixture (the existing `fake_context` sets `meta["root"] == context_dir.parent` and cannot exercise the nested case) carrying a **real `.git`** toplevel (or monkeypatching `is_git_repo`/`resolve_git_dir`); the equality check anchors to the **resolved** git toplevel, not string-equality.

**Reality-check atomicity + reporting**
- [ ] The confidence mirror stages both `feature.json` and `INDEX.json` payloads, then replaces back-to-back via `write_text_atomic`, **mirroring INDEX before popping the stash** so a failure can't lose it (the current code pops first — `confidence.py:90`); a raise **before the first replace** (monkeypatch **`Path.replace`**, not `os.replace`) leaves both files unchanged (fault-injection test). The residual single-replace window is **documented as best-effort** (INDEX may lag, reconciled next run) — not claimed crash-atomic. A "no INDEX entry matched" case (the zero-match loop at `:108-110`) surfaces a signal, not a silent no-op. Both files dumped `sort_keys=True`.
- [ ] `reality-check --demote` prints one line per transition — `demoted X→AMBIGUOUS` / `restored …→Y` / `unchanged` — emitted at the **CLI**. This requires **widening `demote_feature_on_contradiction`/`promote_feature_on_clean` from their current bare `bool` to `(transition, from_value, to_value) | None`** (the bool cannot carry the prior value or distinguish the three transitions; `render_report_md(report)` alone cannot see them). Proven by a `capsys` test asserting the substring for all three transitions. `_reality-check.json` written `sort_keys=True`.

**cli-dispatch UX**
- [ ] `query "x" --bogus 5` (unknown flag) routes through `usage_error` (exit 2, `--help` hint) instead of folding into the search string, and `query`'s existing exit-2 sites are routed through `usage_error`; the no-match exit-1 is documented in `query`'s USAGE — proven by tests. **(The trailing `query "x" --top-k` guard already shipped in `c574d41` — `query.py:60-68` — cover it with a regression test, do not re-implement.)**

**Perf + maintainability**
- [ ] P2: in one `build_all` over a sample repo, each `.py`/`.java` file's content is read **at most twice**, counted by a **path-keyed `defaultdict[Path,int]`** (asserting `counts[f] <= 2` per source path) — a method-level spy cannot discriminate, since the extractor read (`generic.py:42`), the cache-hash read (`cache.py:38`), and the excluded `--changed` hash (`incremental.py:341`) all route through the same `Path.read_bytes`; "exclude the `--changed` hash" means a re-hash of an already-counted path must not push it past 2. (Alternatively Task 12 introduces one named `read_source_bytes(path)` seam all three consumers call, giving the spy a single wrap point.) A file mutated on disk *after* its cache entry is written yields one consistent byte-state (no stale-node/new-byte mix). Output `symbol-graph.json` byte-identical before/after (golden).
- [ ] `latest_status` scanning logic exists once as `last_matching` in `context/domains/log_scan.py` (no leading-underscore filename); `audit/log.py` and `council.py` call it; behaviour byte-identical (existing resumption tests pass). The two "deliberately NOT extracted" NOTE blocks (`audit/log.py:169-176`, `council.py:337-344`) are **quoted, rebutted, and removed** (rebuttal: `atomic_io.py` is the sanctioned shared-domain-helper precedent `folder-organization.md:69-73` prescribes).
- [ ] Full suite green (`python -m pytest tests/ -q`); all new tests pass.

## Open questions

> **GATE DECISION (2026-06-23, recorded at build time): strategy (b) — always-Louvain for committed bytes.** The committed `community` field is produced by NetworkX-Louvain only (pure-Python, always present); no new hard dependency on graspologic. Leiden, when installed, may still serve non-committed/on-demand use. Test note (per F6/F7): the "Leiden never called" spy is **vacuous when graspologic is absent**, so the load-bearing acceptance is **byte-identical `community` across two consecutive builds** (constructible in this venv); the spy assertion is only meaningful with graspologic installed + `leiden` monkeypatched and is left as a CI-gated extra.

- [x] **GATE — clustering backend determinism strategy.** `analysis/cluster.py` falls back graspologic-Leiden → NetworkX-Louvain; the two produce different partitions, so committed `community` is only cross-machine-stable when backend presence is constant. There is **no injectable backend seam today** (`_partition` does `from graspologic.partition import leiden` inside `try/except ImportError`), so the test must be **single-backend** per strategy (cross-backend equality is false by construction). Pick: **(a)** require graspologic (committed always-Leiden; breaks installs relying on the fallback) — *(a)'s fail-fast half is the most test-robust given graspologic is **absent** in the venv*; **(b)** always-Louvain for committed bytes (drop Leiden from the committed path) — *its "Leiden never called" spy is **vacuous** unless graspologic is installed + `leiden` monkeypatched*; **(c)** exclude `community` from committed `symbol-graph.json` — **a breaking contract change, confirmed**: feature-grouping **does** read the node `community` field — `runner.py:182-187` re-reads the just-written `symbol-graph.json` from disk and feeds it to `builder.py:76` (`community` lives only in the persisted file, never on the in-memory graph — `export/graph.py:29-31`), so (c) additionally requires re-routing `runner`/`builder` to an in-memory partition (unscoped today). Strategies **(a)/(b) keep the field and are safe for this consumer.** All three force a one-time `community` rewrite in this repo's own committed `.context/`; the chosen single-backend test should use a graph large enough to hit the recursive `_split_community` second partition (`cluster.py:113-128`). **Decide before the cluster task is implemented.**
